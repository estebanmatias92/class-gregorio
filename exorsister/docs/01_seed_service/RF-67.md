# RF-67: Determinismo e Idempotencia

| Campo | Valor |
|-------|-------|
| **ID** | RF-67 |
| **Módulo** | Servicio de Semilla (SeedService) |
| **Prioridad** | Alta |
| **Origen** | CU-01 |
| **Dependencias** | RF-66 |

## Descripción

El sistema debe garantizar que, para una misma semilla y los mismos parámetros de consulta (ej. día, ubicación), la respuesta sea siempre idéntica en todas las iteraciones de la partida.

## Criterios de aceptación

- Cada consulta del servicio es una función pura del `seedId` y sus parámetros: mismos inputs → mismo output en toda ejecución.
- No existe estado mutable ni aleatoriedad en las consultas del servicio.
- Cambiar un solo parámetro puede producir un resultado distinto (no se exige que sea distinto, pero no se cachea el resultado anterior).
- Aplica a todas las consultas del servicio (RF-66).

## Backlog asociado

### S-09: Garantizar determinismo e idempotencia

- **Prioridad:** Alta
- **Dependencias:** S-02 al S-08
- **Descripción:** Garantizar que para una misma semilla y los mismos parámetros de consulta (ej. día, ubicación), la respuesta sea siempre idéntica en todas las iteraciones de la partida. Esto aplica a todas las funciones de consulta del servicio.
- **Criterios de aceptación:**
  - Ejecutar cualquier consulta con los mismos parámetros N veces produce exactamente el mismo resultado.
  - Cambiar un solo parámetro puede producir resultado distinto (no se exige que sea distinto, pero no se cachea el resultado anterior).

### S-10: Tests de idempotencia

- **Prioridad:** Alta
- **Dependencias:** S-09
- **Descripción:** Suite de tests que verifique la idempotencia de todas las consultas del servicio. Cada test ejecuta la misma consulta múltiples veces y verifica que el resultado sea idéntico.
- **Criterios de aceptación:**
  - Cada función de S-02 a S-08 tiene al menos un test de idempotencia.
  - Los tests usan al menos 3 semillas diferentes y 3 conjuntos de parámetros distintos.
