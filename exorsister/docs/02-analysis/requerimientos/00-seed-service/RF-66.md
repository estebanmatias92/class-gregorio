# RF-66: Servicio Central de Semilla

| Campo | Valor |
|-------|-------|
| **ID** | RF-66 |
| **Módulo** | Servicio de Semilla (SeedService) |
| **Prioridad** | Alta |
| **Origen** | CU-01 |
| **Dependencias** | Ninguna |

## Descripción

El sistema debe exponer un servicio central que, dada una semilla, responda consultas deterministas para cada módulo: ubicación de eventos, estadísticas de objetos, datos de enemigos, misiones diarias, recompensas, tipo de Mal del Mundo y páginas de ritual.

## Criterios de aceptación

- El servicio expone las siguientes consultas:
  - `getEnemyAt(location, day, hour)` → `EnemyBlueprint`
  - `getItemStats(itemId)` → `ItemStats`
  - `getDailyMission(day)` → `MissionBlueprint`
  - `getEventAt(location)` → `EventBlueprint`
  - `getMalDelMundo()` → `MalType`
  - `getRitualPages()` → `PageBlueprint[]`
  - `getRewardPool(enemyId)` → `Reward[]`
- Todas las consultas reciben el `seedId` como parámetro implícito o explícito.
- Cada consulta retorna el tipo de dato correspondiente o `null` si no existe.

## Backlog asociado

### S-01: generateSeed() — Generar semilla determinista

- **Prioridad:** Alta
- **Dependencias:** Ninguna
- **Descripción:** Implementar función que genere un `seedId` (ej. numérico o string hash) a partir del cual todas las demás consultas sean deterministas. Es el punto de entrada del módulo.
- **Criterios de aceptación:**
  - `generateSeed()` produce siempre el mismo `seedId` si se invoca en idénticas condiciones de inicio.
  - El `seedId` es persistible y puede reutilizarse para reconstruir el mismo estado de partida.

### S-02: getEnemyAt(location, day, hour) — Consultar enemigo por ubicación y tiempo

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId`, una ubicación, un día y una hora, devolver un `EnemyBlueprint` (tipo de enemigo, partes anatómicas, HP, habilidades, orden de turno). Cubre enemigos errantes, territoriales, fantasmales y especiales.
- **Criterios de aceptación:**
  - Mismos parámetros + mismo `seedId` → mismo `EnemyBlueprint`.
  - Si no hay enemigo definido para esa ubicación/horario, devuelve `null`.

### S-03: getItemStats(itemId) — Consultar estadísticas de objeto

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId` y un `itemId`, devolver `ItemStats` (clasificación 0.0–5.0, tipo, estadísticas, habilidades ocultas, etc.). Cada objeto de la partida tiene estadísticas fijadas por la semilla.
- **Criterios de aceptación:**
  - Mismo `seedId` + mismo `itemId` → mismas `ItemStats`.
  - Si el `itemId` no existe en la semilla, devuelve `null`.

### S-04: getDailyMission(day) — Consultar misión del día

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId` y un número de día, devolver `MissionBlueprint` (objetivo, ubicación, pistas de enemigo, recompensa, fecha límite). Las misiones son fijas por semilla: mismo día, misma misión en toda iteración.
- **Criterios de aceptación:**
  - Mismo `seedId` + mismo día → mismo `MissionBlueprint`.
  - Si no hay misión asignada para ese día, devuelve `null`.

### S-05: getEventAt(location) — Consultar evento en ubicación

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId` y una ubicación, devolver `EventBlueprint` (tipo de evento, condiciones de activación, recompensas, consecuencias). Los puntos de evento están fijados por la semilla.
- **Criterios de aceptación:**
  - Mismo `seedId` + misma ubicación → mismo `EventBlueprint`.
  - Si no hay evento en esa ubicación, devuelve `null`.

### S-06: getMalDelMundo() — Consultar tipo de Mal del Mundo

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId`, devolver `MalType` (ej. Mal Carmesí, Mal de Dinero, Mal de Conocimiento). La variante del Mal del Mundo es fijada por la semilla al inicio de la partida.
- **Criterios de aceptación:**
  - Mismo `seedId` → mismo `MalType` en toda la partida.

### S-07: getRitualPages() — Consultar páginas de ritual

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId`, devolver `PageBlueprint[]` (conjunto de páginas de ritual disponibles en la semilla, cada una con su ubicación y pista).
- **Criterios de aceptación:**
  - Mismo `seedId` → mismo conjunto de páginas.
  - Las páginas no se repiten dentro de la misma semilla.

### S-08: getRewardPool(enemyId) — Consultar recompensas de enemigo

- **Prioridad:** Alta
- **Dependencias:** S-01
- **Descripción:** Dado un `seedId` y un `enemyId`, devolver `Reward[]` (dinero, objetos, páginas, etc.) que otorga ese enemigo al ser derrotado.
- **Criterios de aceptación:**
  - Mismo `seedId` + mismo `enemyId` → misma `Reward[]`.
  - Si el enemigo no tiene recompensa definida, devuelve lista vacía.
